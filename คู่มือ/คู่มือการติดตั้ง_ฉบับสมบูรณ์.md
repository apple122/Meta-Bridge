# 🛠️ คู่มือการติดตั้ง Meta-Bridge (ฉบับสมบูรณ์ที่สุด)
**Version:** Master 4.2 (Trade Control & Audit System)
**วันที่อัปเดตล่าสุด:** 22 เมษายน 2026

คู่มือนี้ฉบับเดียวรวมทุกขั้นตอนการติดตั้งตั้งแต่ศูนย์จนพร้อมใช้งาน 100% ทั้งระบบเทรด, ระบบควบคุมผลกำไร, และระบบจัดการวอลเล็ทฉบับปรับปรุง

---

## 1. การเตรียมความพร้อม (Prerequisites)
ก่อนเริ่มติดตั้ง กรุณาตรวจสอบว่าเครื่องของคุณมี:
- **Node.js**: เวอร์ชัน 18.x หรือสูงกว่า (แนะนำ 20.x+)
- **npm**: เวอร์ชันล่าสุด

---

## 2. การตั้งค่าสภาพแวดล้อม (Environment Setup)
สร้างไฟล์ `.env` ที่โฟลเดอร์หลัก (Root) ของโปรเจกต์ และใส่ข้อมูลตั้งค่าดังนี้:

```env
# 1. Supabase Connection (จาก Project Settings > API)
VITE_SUPABASE_URL=ของคุณ_supabase_url
VITE_SUPABASE_ANON_KEY=ของคุณ_supabase_anon_key

# 2. Security & Encryption
VITE_APP_SECRET_KEY=meta_bridge_secure_vault_xyz_123
```

---

## 3. การตั้งค่าฐานข้อมูล (Database Setup) 🗄️
ขั้นตอนนี้สำคัญที่สุดเพื่อให้ระบบเทรดและระบบจัดการแอดมินทำงานได้ 100%

### สำหรับการติดตั้งครั้งแรก (New Install):
1. ไปที่ **Supabase Dashboard > SQL Editor**
2. คัดลอกเนื้อหาทั้งหมดจากไฟล์: `src/lib/supabase_schema.sql` มาวางและกด **Run**

### สำหรับการอัปเดตระบบเดิม (Incremental Update):
หากคุณมีฐานข้อมูลอยู่แล้วและต้องการอัปเดตฟีเจอร์ **Trade Control** และ **Wallet Audit**:
1. นำโค้ดจากไฟล์ `src/lib/TRADE_CONTROL_MIGRATION.sql` ไปรันใน SQL Editor
2. คำสั่งนี้จะเพิ่มฟิลด์ควบคุมกำไรและประวัติธุรกรรมโดยไม่ลบข้อมูลเก่าครับ

---

## 4. ติดตั้งและเริ่มรันโปรเจกต์ (Installation)
เปิด Terminal ในโฟลเดอร์โปรเจกต์แล้วใช้คำสั่งดังนี้:

```bash
# 1. ติดตั้งไลบรารีที่จำเป็น
npm install

# 2. เริ่มรันแอปพลิเคชัน
npm run dev
```
เข้าใช้งานผ่าน: `http://localhost:5173`

---

## 5. การตั้งค่าผู้ดูแลระบบ (Admin Setup)
1. สมัครสมาชิกผ่านหน้าเว็บปกติ
2. ไปที่ **Supabase > Table Editor > profiles**
3. ค้นหา User ของคุณ และแก้ไขคอลัมน์ `role` ให้เป็น `'admin'` (เดิมคือ `is_admin`)
4. Logout และ Login ใหม่เพื่อเข้าใช้งานระบบ Admin

---

## 6. ระบบควบคุมการเทรด (Trade Control) ⚙️
แอดมินสามารถกำหนดผลการเทรดรายบุคคลได้ที่หน้า **Admin Dashboard > Users > Edit Control**:
*   **Normal**: ตามราคาตลาดจริง
*   **Always Win**: ล็อกผลชนะเสมอ
*   **Always Loss**: ล็อกผลแพ้เสมอ
*   **Low Win Rate**: สุ่มชนะเพียง 15% (เหมาะสำหรับโหมดความยากสูง)

---

## 7. การแจ้งเตือนและอีเมล (Email & Edge Functions) 📧
ระบบส่งเมลผ่าน **Supabase Edge Functions** ร่วมกับ **Brevo API**:

1.  **Deploy Functions**:
    ```bash
    npx supabase functions deploy send-email --no-verify-jwt
    npx supabase functions deploy resolve-trades --no-verify-jwt
    ```
2.  **ตั้งค่า Secrets**: นำ API Key จาก Brevo ไปใส่ใน Supabase Dashboard หรือรันคำสั่ง:
    ```bash
    npx supabase secrets set BREVO_API_KEY=your_api_key
    ```

---

## 💡 การแก้ไขปัญหา (Troubleshooting)
*   **ประวัติธุรกรรมไม่ขึ้น:** ตรวจสอบว่าได้รัน SQL Migration เพื่อเพิ่มคอลัมน์ `description` ในตาราง `transactions` หรือยัง
*   **เทรดแล้วผลไม่ตรง:** ตรวจสอบค่า `trade_control` ในตาราง `profiles` ว่าเผลอไปตั้งค่าล็อกผลไว้หรือไม่
*   **อีเมลไม่ส่ง:** ตรวจสอบ `BREVO_API_KEY` ใน Supabase Secrets และตรวจสอบ Sender Email ใน Brevo ว่า Verified หรือยัง
