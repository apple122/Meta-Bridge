# 🛠️ คู่มือการติดตั้ง Meta-Bridge (ฉบับสมบูรณ์ที่สุด)
**Version:** Master 4.1 (Performance & Admin Dashboard)
**วันที่อัปเดตล่าสุด:** 12 เมษายน 2026

คู่มือนี้ฉบับเดียวรวมทุกขั้นตอนการติดตั้งตั้งแต่ศูนย์จนพร้อมใช้งาน 100% ทั้งระบบเทรด, ระบบความปลอดภัย, และระบบแจ้งเตือน

---

## 1. การเตรียมความพร้อม (Prerequisites)
ก่อนเริ่มติดตั้ง กรุณาตรวจสอบว่าเครื่องของคุณมี:
- **Node.js**: เวอร์ชัน 18.x หรือสูงกว่า (แนะนำ 20.x+)
- **npm**: เวอร์ชันล่าสุด

---

## 2. การตั้งค่าสภาพแวดล้อม (Environment Setup)
สร้างไฟล์ `.env` ที่โฟลเดอร์หลัก (Root) ของโปรเจกต์ และใส่ข้อมูลตั้งค่าดังนี้:

```env
# 1. Supabase Connection (จาก Project Settings > API)
VITE_SUPABASE_URL=ของคุณ_supabase_url
VITE_SUPABASE_ANON_KEY=ของคุณ_supabase_anon_key

# 2. Security & Encryption
# (คีย์สำหรับเข้ารหัสรหัสผ่านและการเข้าถึงข้อมูล)
VITE_APP_SECRET_KEY=meta_bridge_secure_vault_xyz_123

# 3. Email Settings (EmailJS สำหรับระบบ OTP)
VITE_EMAILJS_SERVICE_ID=ของคุณ_service_id
VITE_EMAILJS_TEMPLATE_ID=ของคุณ_otp_template_id
VITE_EMAILJS_PUBLIC_KEY=ของคุณ_public_key

# 4. specialized Email Templates (Optional)
VITE_EMAILJS_TEMPLATE_DEPOSIT_ID=ของคุณ_deposit_template_id
VITE_EMAILJS_TEMPLATE_WIN_ID=ของคุณ_win_template_id
```

---

## 3. การตั้งค่าฐานข้อมูล (Database Setup)
ขั้นตอนนี้สำคัญที่สุดเพื่อให้ระบบเทรดและ Login ทำงานได้ 100%

1. ไปที่ **Supabase Dashboard > SQL Editor**
2. สร้าง **New Query**
3. คัดลอกเนื้อหาทั้งหมดจากไฟล์: `src/lib/supabase_schema.sql` มาวางในหน้าต่าง Query
4. กดปุ่ม **Run**
   > [!IMPORTANT]
   > คำสั่งนี้จะล้างข้อมูลเก่าและสร้างตารางรวมถึงฟังก์ชันการเทรด (RPC) ทั้งหมดให้ใหม่โดยอัตโนมัติ

---

## 4. ติดตั้งและเริ่มรันโปรเจกต์ (Installation)
เปิด Terminal ในโฟลเดอร์โปรเจกต์แล้วใช้คำสั่งดังนี้:

```bash
# 1. ติดตั้งไลบรารีที่จำเป็น
npm install

# 2. เริ่มรันแอปพลิเคชัน
npm run dev
```
เข้าใช้งานผ่าน: `http://localhost:5173`

---

## 5. การตั้งค่าผู้ดูแลระบบ (Admin Setup)
1. สมัครสมาชิกผ่านหน้าเว็บปกติ
2. ไปที่ **Supabase > Table Editor > profiles**
3. ค้นหา User ของคุณ และแก้ไขคอลัมน์ `is_admin` ให้เป็น `true`
4. Logout และ Login ใหม่เพื่อเข้าใช้งานระบบ Admin

---

## 6. การตั้งค่าระบบแจ้งเตือน (Push Notifications)
เมื่อเข้าหน้าเว็บครั้งแรก บราวเซอร์จะขออนุญาตแจ้งเตือน
1. กด **"Allow" (อนุญาต)** เพื่อรับการแจ้งเตือนความปลอดภัย
2. ระบบจะบันทึกเครื่องของคุณลงฐานข้อมูลโดยอัตโนมัติ

---

## 7. การตั้งค่า EmailJS Templates ด้วยไฟล์ HTML 📧
โปรเจกต์นี้มีไฟล์ HTML Template ที่ออกแบบไว้อย่างสวยงามพร้อมใช้งานในโฟลเดอร์ `email-templates/` คุณไม่จำเป็นต้องจัดหน้าใหม่ เพียงแค่คัดลอกโค้ดไปวางใน EmailJS ได้เลย

### ขั้นตอนการนำไปใช้:
1. ไปที่เว็บไซต์ **EmailJS.com** > เข้าสู่ระบบ > ไปที่เมนู **Email Templates**
2. กดปุ่ม **Create New Template**
3. ในส่วนของเนื้อหา (Content) ให้สลับเป็นโหมด **Source Code (`</>`)** 
4. เปิดไฟล์เป้าหมายจากโฟลเดอร์ `email-templates/` ในโปรเจกต์ คัดลอกโค้ดทั้งหมดไปวางทับใน EmailJS แล้วกด **Save**
5. นำ Template ID ที่ได้ ไปกำหนดในไฟล์ `.env`

### เทมเพลตที่มีให้และ `.env` ที่เกี่ยวข้อง:

#### 1. ระบบยืนยันตัวตน (OTP Verification)
*   **ไฟล์ที่ต้องใช้:** `email-templates/template_otp.html`
*   **ตั้งค่า `.env`:** `VITE_EMAILJS_TEMPLATE_ID`
*   **คำอธิบาย:** ใช้สำหรับการสมัครสมาชิก, รีเซ็ตรหัสผ่าน, เปลี่ยนอีเมล

#### 2. แจ้งเตือนการทำธุรกรรม (Deposit / Withdraw Notification)
*   **ไฟล์ที่ต้องใช้:** `email-templates/template_deposit.html`
*   **ตั้งค่า `.env`:** `VITE_EMAILJS_TEMPLATE_DEPOSIT_ID`
*   **คำอธิบาย:** แจ้งเตือนลูกค้าเมื่อมีการทำรายการฝาก-ถอนสำเร็จ

#### 3. แจ้งเตือนเมื่อชนะการเทรด (Win Celebration)
*   **ไฟล์ที่ต้องใช้:** `email-templates/template_win.html`
*   **ตั้งค่า `.env`:** `VITE_EMAILJS_TEMPLATE_WIN_ID`
*   **คำอธิบาย:** ระบบจะส่งเมลแจ้งเงินรางวัลพร้อมรายละเอียดการเข้าทำกำไร (มีตัวแปร `{{ticket_id}}` เพิ่มเข้ามา)

### 💡 ตัวแปร (Variables) ในระบบ
ระบบในไฟล์ `src/services/emailService.ts` จะจัดการส่งค่าเหล่านี้เข้าเทมเพลตให้อัตโนมัติ:
*   `{{to_name}}`: Account name of the recipient
*   `{{email_subject}}` / `{{EMAIL_SUBJECT}}`: Subject of the email (localized)
*   `{{email_title}}` / `{{EMAIL_TITLE}}`: Title shown in the header
*   `{{email_greeting}}` / `{{EMAIL_GREETING}}`: Hello greeting
*   `{{email_body}}` / `{{EMAIL_BODY}}`: The main message content
*   `{{email_otp_label}}` / `{{EMAIL_OTP_LABEL}}`: Label for the passcode/amount
*   `{{passcode}}`: OTP code or transaction amount
*   `{{time}}`: Timestamp or OTP expiry
*   `{{ticket_id}}`: Trade Order ID (Win template only)
*   **Note:** The system supports both **lowercase** and **UPPERCASE** variants for template flexibility.

## 💡 การแก้ไขปัญหา (Troubleshooting)
*   **เทรดไม่ได้ (RPC Error):** ตรวจสอบว่าได้รัน SQL ในขั้นตอนที่ 3 ครบถ้วนหรือไม่ (ต้องมีฟังก์ชัน `place_binary_trade`)
*   **เข้าหน้า Admin ไม่ได้:** เช็คตาราง `profiles` ว่า `is_admin` เป็น `true` หรือยัง
*   **Error 400 ยิบย่อย:** แนะนำให้รัน SQL จากไฟล์ `supabase_schema.sql` ใหม่อีกครั้งเพื่อเริ่มฐานข้อมูลใหม่ที่สะอาด 100%

## 8. การใช้งานระบบตัดสินผลอัตโนมัติ (Background Resolution) ⚙️
เพื่อให้ระบบตัดสินผลการเทรดและส่งอีเมลแจ้งเตือนได้ตลอด 24 ชั่วโมง แม้คุณจะไม่ได้เปิดหน้าเว็บทิ้งไว้:

### 1. การ Deploy Edge Function
ใน Terminal ของโปรเจกต์ รันคำสั่งนี้:
```bash
npx supabase functions deploy resolve-trades --no-verify-jwt
```

### 2. ตั้งค่าตัวแปรใน Supabase Dashboard
ไปที่ **Settings > Edge Functions** แล้วเพิ่ม Secret ดังนี้:
*   `EMAILJS_SERVICE_ID`
*   `EMAILJS_PUBLIC_KEY`
*   `EMAILJS_PRIVATE_KEY` (สำคัญมาก: สำหรับใช้ส่งเมลจากหลังบ้าน)
*   `EMAILJS_TEMPLATE_WIN_ID`
*   `SUPABASE_URL` และ `SUPABASE_SERVICE_ROLE_KEY` (ปกติจะมีให้อยู่แล้ว)

### 3. เปิดการทำงานอัตโนมัติ (Cron Job)
ไปที่ **SQL Editor** และรันสคริปต์ท้ายไฟล์ `src/lib/supabase_schema.sql` (ส่วนของ STEP 7) เพื่อให้เซิร์ฟเวอร์คอยตัดสินผลให้ทุก 1 นาที
