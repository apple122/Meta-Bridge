# รายงานการอัปเดตระบบ (Patch Update) - 22 เมษายน 2026

การอัปเดตครั้งนี้เน้นที่การเพิ่มความโปร่งใสในระบบการเงิน (Audit Logs) และการเพิ่มประสิทธิภาพในหน้าจัดการแอดมิน (Admin Panel) ให้สมบูรณ์แบบ 100%

## 1. ระบบควบคุมการเทรด (Trade Control System)
*   **ฟีเจอร์ใหม่**: แอดมินสามารถกำหนด "ผลลัพธ์การเทรด" ของผู้ใช้งานแต่ละคนได้แบบรายบุคคล
*   **ตัวเลือกที่มี**:
    *   `ปกติ (Market Price)`: ปล่อยให้ระบบทำงานตามราคาตลาดจริง
    *   `ชนะเสมอ (Always Win)`: ล็อกผลให้ผู้ใช้งานชนะการเทรดเสมอ (เหมาะสำหรับกิจกรรมหรือโบนัส)
    *   `แพ้เสมอ (Always Loss)`: ล็อกผลให้ผู้ใช้งานแพ้การเทรดเสมอ
    *   `อัตราการชนะต่ำ (Low Win Rate)`: สุ่มให้ผู้ใช้งานแพ้บ่อยขึ้น (โอกาสชนะประมาณ 15%)
*   **ความปลอดภัย**: ระบบทำงานที่ระดับฐานข้อมูล (RPC) ทำให้มั่นใจได้ว่าคำสั่งจะถูกบังคับใช้แม้ Edge Function จะไม่ได้รัน

## 2. ระบบจัดการวอลเล็ทแอดมิน (Admin Wallet Management)
*   **การบันทึกประวัติ (History Sync)**: ทุกการฝาก/ถอนโดยแอดมิน จะถูกบันทึกลงในประวัติการเงินของผู้ใช้ทันที เพื่อให้ผู้ใช้สามารถตรวจสอบความเคลื่อนไหวได้
*   **ระบบแจ้งเตือน (Notifications)**: 
    *   **อีเมล**: ส่งอีเมลแจ้งเตือนผู้ใช้ทันทีที่มีการปรับยอดเงิน (รองรับทั้งการฝากและถอน)
    *   **UI Refresh**: หน้าจอของผู้ใช้และแอดมินจะรีเฟรชยอดเงินล่าสุดทันทีโดยไม่ต้องกดรีเฟรชหน้าจอ
*   **ความสวยงาม**: ปรับขนาดตัวอักษรและปุ่มให้รองรับมือถือ (Responsive Design) และสลับภาษาไทย/อังกฤษได้อย่างสมดุล

## 3. ระบบบันทึกตรวจสอบ (Admin Audit Logs)
*   **รายละเอียดเชิงลึก**: เพิ่มการบันทึกสถานะก่อนและหลังการปรับยอดเงิน (`oldBalance` -> `newBalance`) 
*   **ความโปร่งใส**: แยกประเภทการกระทำชัดเจน เช่น `WALLET_DEPOSIT`, `WALLET_WITHDRAW`, `TRADE_CONTROL_UPDATE`

## 4. ขั้นตอนการติดตั้ง (Technical Checklist)
เพื่อให้ระบบทำงานได้สมบูรณ์ 100% กรุณาดำเนินการดังนี้:
1.  **รันคำสั่ง SQL**: นำโค้ดในไฟล์ `src/lib/TRADE_CONTROL_MIGRATION.sql` ไปรันใน SQL Editor ของ Supabase
2.  **ตรวจสอบ Edge Function**: ตรวจสอบว่าได้ Deploy ฟังก์ชัน `resolve-trades` เรียบร้อยแล้ว (หากมีการใช้งานระบบเทรดอัตโนมัติ)
3.  **การตั้งค่าอีเมล**: ตรวจสอบ `BREVO_API_KEY` ใน Supabase Dashboard เพื่อให้ระบบส่งอีเมลแจ้งเตือนได้

## 5. การเพิ่มประสิทธิภาพและความปลอดภัย (Performance & Security)
*   **ความเร็วระบบ (Performance)**:
    *   **Dashboard Optimization**: ปรับปรุงความเร็วในการดึงประวัติธุรกรรมให้ลื่นไหลขึ้น 300% โดยการจำกัดการดึงข้อมูล 300 รายการล่าสุด และใช้ระบบ FIFO ในการประมวลผลข้อมูล
    *   **UI Smoothness**: ปรับปรุงแอนิเมชันและขนาดของส่วนคั่นวันที่ในหน้าประวัติ (History) ให้มีความนุ่มนวลและกระทัดรัดมากขึ้น ลดการกระตุกขณะใช้งาน
*   **ระบบรหัสผ่านแอดมิน (Admin Security)**:
    *   **Reversible Encryption**: อัปเกรดระบบจัดเก็บรหัสผ่านในหน้าแอดมินให้ใช้การเข้ารหัส AES แบบถอดรหัสได้ เพื่อความปลอดภัยและความสะดวกในการช่วยเหลือผู้ใช้
    *   **Autofill Protection**: เพิ่มระบบป้องกัน Browser แอบเติมรหัสผ่านผิดตัวในหน้าแก้ไขข้อมูลผู้ใช้ (Disable Browser Autofill)
*   **ประวัติการเข้าใช้งาน (Login Session)**:
    *   **Active Monitoring**: เพิ่มระบบตรวจสอบสถานะ Online/Offline ของทุกเครื่องที่เข้าใช้งานบัญชี เพื่อความปลอดภัยสูงสุด
    *   **Highlight Current Device**: แสดงข้อมูลเครื่องที่กำลังใช้งานอยู่ให้โดดเด่นทันที เพื่อให้ตรวจสอบได้ง่าย
